# خريف ظفار 2026 — Khareef Dhofar PWA

تطبيق ويب تقدّمي (PWA) كامل — جاهز للنشر والتثبيت على الشاشة الرئيسية.

## ما الذي يحتويه هذا المشروع؟

```
.
├── index.html              ← وسوم PWA (manifest, theme-color, أيقونات)
├── manifest.json (public/) ← اسم التطبيق، الألوان، الأيقونات
├── service-worker.js (public/) ← يجعل التطبيق يعمل أوفلاين ويُمكِّن التثبيت
├── public/icons/            ← أيقونات بأحجام 192px / 512px / maskable
├── src/App.jsx              ← كود التطبيق الكامل (الذي بنيناه سابقًا)
├── src/main.jsx             ← نقطة الدخول + تسجيل الـ Service Worker
└── tailwind.config.js, postcss.config.js ← إعداد Tailwind
```

## التشغيل محليًا

```bash
npm install
npm run dev
```

افتح الرابط الذي يظهر (عادة `http://localhost:5173`).

> ملاحظة: الـ Service Worker وميزة "إضافة إلى الشاشة الرئيسية" **لا تعملان على `localhost` بشكل كامل في كل المتصفحات** — تحتاج نشرًا حقيقيًا على HTTPS لتجربتها فعليًا على الموبايل (الخطوة التالية).

## النشر المجاني على Vercel (الأسهل)

1. أنشئ حسابًا مجانيًا على [vercel.com](https://vercel.com) (يمكن الدخول مباشرة بحساب GitHub).
2. ارفع هذا المجلد إلى مستودع جديد على GitHub (أو استخدم Vercel CLI مباشرة بدون GitHub):
   ```bash
   npm i -g vercel
   vercel
   ```
3. اتبع التعليمات التي تظهر في الطرفية (اختر الإعدادات الافتراضية — Vercel يكتشف Vite تلقائيًا).
4. بعد انتهاء النشر ستحصل على رابط حقيقي مثل:
   `https://khareef-dhofar.vercel.app`

## النشر المجاني على Netlify (بديل)

```bash
npm run build
```
ثم اسحب مجلد `dist/` الناتج وأفلِته في [app.netlify.com/drop](https://app.netlify.com/drop).

## بعد النشر — جرّب التثبيت

1. افتح الرابط من **متصفح Chrome على أندرويد** → سيظهر بانر "تثبيت الآن" الذي بنيناه داخل التطبيق فعليًا الآن.
2. افتح الرابط من **Safari على آيفون** → اضغط زر المشاركة ⬆️ ثم "إضافة إلى الشاشة الرئيسية".
3. تأكد أن الأيقونة التي تظهر على الشاشة الرئيسية هي شعار التطبيق (الغيمة الخضراء) وأن فتحه يعمل بدون شريط عنوان المتصفح (Standalone Mode).

## كيف تُحدّث المحتوى لاحقًا؟

عدّل البيانات مباشرة داخل `src/App.jsx` (المهرجانات، المواقع، الأسعار...)، ثم:
```bash
git add .
git commit -m "تحديث المواعيد"
git push
```
Vercel/Netlify ستعيد النشر تلقائيًا خلال ثوانٍ — والمستخدمون الذين فتحوا التطبيق سابقًا سيحصلون على النسخة الجديدة تلقائيًا في فتحة الإنترنت القادمة (بفضل استراتيجية "network-first" في الـ Service Worker).

## تنبيه مهم

- **لا يوجد "تحديث ذاتي" بدون تدخل بشري** — يحتاج التطبيق دائمًا أن يُحدِّث شخص ما البيانات (المواعيد، الأسعار...) داخل الكود أو ملف بيانات خارجي.
- أيقونات هذا المشروع مولّدة برمجيًا بتصميم بسيط (غيمة) بألوان التطبيق — يمكنك استبدالها بشعار احترافي حقيقي في `public/icons/` بنفس الأسماء والأحجام.
